Tools & Technology

Procreate

A good application not only depend on the code it also requires a good UI design, an interesting UI will be more attractive to users with several UI design skills will improve the using experiences. Most of the members in our group are using IOS and MAC system then we decided to use Procreate as our draft drawing application. Procreate is a drawing application which especially design for iPad. Compared to paper drawing, firstly, Procreate is more efficient, user can easily erase and edit the error without destroy the original piece. Secondly, the Procreate contain various drawing tools pack and various colors to choose. (Gabs Art Tips, 2020) Thirdly, Procreate can save the drafts it is safer than storing the papers. On the other hand, Procreate can easily share the draft with other members in the group and others can edit the draft online. To use this app, we will have to purchase from the App Store at USD 9.99. To have the best outcome from the app, we will use another hardware, Apple Pencil to make our sketches or low fidelity prototype. Because most of us are not in Melbourne currently after using Procreate can achieve remote working.

Figma

Figma is wonderful application for us to arrange the UI also suitable for a small team like us, Figma is like simulator for testing the front-end of an application. We choose Figma because one of the best functions of Figma is it supporting the online editing for all the group members. It reveals the process of designing and save the costs of communication. Compared to Sketch, Figma is not only a UI designing tool, Figma combine Invision, Zeplin, Dropbox, liveshare and Overflow. It likely covers most of the area of designing digital products. (Kopf, 2018) An example for how efficient the Figma is, the simplest process without using Figma is: Firstly, use Sketch finish the designing of the UI, secondly upload the draft to Invision to make interactive prototype, finally upload to Zeplin to hand over to the engineer, however, all these steps can be done within Figma. Normally, we would have to pay subscription fees, however, RMIT University provides access to Figma to the students. Using both tools, we will develop a prototype of the application.

Machine Learning

To implement the machine learning function for Foodie (learn and analyze the food habit the favor of the user.) We decided to use TensorFlow for accomplish machine learning in Foodie, TensorFlow is mathematic system based on the dataflow programming. It is commonly using in machine learning the former of it is DisBelief of Google. TensorFlow has multi-layer structure can be place in websites, PC clients and servers. It supports the high computing from GPU and TPU. (www.tutorialspoint.com, n.d.) The reason for us to choose TensorFlow:

1. The usability and adaptability (www.javatpoint.com, n.d.)

The working process of TensorFlow is kind of simple compared to other machine learning system. It has a stable API, good compatibility and very fast compile speed which allows programmer to iterate easier and smoother.

2. The flexibility (www.javatpoint.com, n.d.)

TensorFlow can be use in different devices, it can be run in multiple GPU and CPU. Also, TensorFlow support Java so that we can implement it into our application.

3. The supporting from the community

TensorFlow was developed by Google, and it aim to be commonly used by programmers, besides, Google also use it for their program and keep provide supports for it. Therefore, TensorFlow formed a powerful community, the resources in the community may be helpful for our team to use.
